# Layover Website — Project Proposal

## Concept

A web tool that helps travelers make the most of a long layover in an unfamiliar city. Enter your layover airport and duration, and get a timed, realistic mini-itinerary with transit time already baked in — so you can leave the airport confidently.

---

## Problem

Long layovers (4–12 hours) are mostly wasted sitting in airports. Most travelers don't leave because they don't know:
- How long it takes to get into the city and back
- What's actually worth seeing in the time they have
- Whether they'll make it back in time

---

## Target Users

- Frequent flyers routing through major hub cities
- Backpackers and budget travelers on long-haul routes
- Anyone transiting through cities like Dubai, Istanbul, Singapore, Frankfurt, Amsterdam, Tokyo, and similar hubs

---

## Core Feature

**Input:** Layover airport + total layover duration (hours)

**Output:** A timed mini-itinerary that includes:
- Transit time to/from the city (both directions)
- Airport buffer time (check-in, security)
- 2–4 recommended stops with realistic time at each
- A clear "leave by" time to make it back safely

---

## Why It Works

Itineraries are generated by Claude AI, which can reason about:
- Real transit options from each airport (train, metro, taxi)
- Neighborhood proximity and walkability
- Time-of-day considerations (e.g. rush hour, opening hours)
- Visa-free transit eligibility for common passport holders

---

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | React 19 + Next.js 15 |
| Styling | Tailwind CSS v4 + Shadcn UI |
| Icons | Lucide React |
| AI | Claude API (server-side via Next.js API routes) |
| Language | TypeScript |

**Why Next.js over plain Vite:** The Claude API key must be kept server-side. Next.js provides built-in API routes so the key is never exposed to the browser, without needing a separate backend service.

---

## Pages / Views

### Home / Input
- Airport selector (searchable by city or IATA code)
- Layover duration picker (4h, 5h, 6h, 8h, 10h, 12h+)
- Optional: departure terminal, nationality (for visa notes)
- Generate button

### Itinerary View
- Timeline layout showing each stop with:
  - Time window (e.g. 10:15 – 11:00)
  - Location name + one-line description
  - Transit method to next stop
- "Leave airport by" and "Return by" prominently displayed
- Print / share option
